# Atlas 迁移到新电脑

## 1. 完整解压

把整个压缩包解压到一个会长期保留的目录。路径可以包含中文，但不要直接在 ZIP 内运行，也不要只复制 `index.html`；页面、`assets`、`data`、`scripts` 与 `.codex` 必须保持原有相对位置。

ZIP 不包含原电脑的 Codex 登录信息、API key、浏览器会话、模型权限、额度或自动任务。接收者必须使用自己的账号和环境。

## 2. 启动静态网站

新电脑安装 Python 3 后，双击 `启动图鉴.bat`。浏览器应打开：

```text
http://127.0.0.1:8000/index.html
```

项目有且只有 10 个正式页面：`index / explore / model / compare / radar / venues / lineage / timeline / trends / glossary`，不包含学习页、Quiz 或答题页。学术追踪固定有三个可分享视图：

```text
venues.html?view=journals
venues.html?view=conferences
venues.html?view=compare
```

手机需要与电脑连接同一 Wi-Fi。启动窗口会显示当次真实局域网地址；IP 可能在重新联网后变化，不要沿用旧电脑的地址。若 Windows 防火墙阻挡，可双击 `允许手机访问.bat`，它只开放本地子网到 TCP 8000。电脑本机或模拟手机视口通过不等于真机通过，必须实际用同一 Wi-Fi 的手机打开页面后才能写“真机已验证”。

## 3. 学术追踪的数据边界

当前 `data/academic_tracker.json` 包含 8 个期刊入口、10 个会议入口和 10 条人工核验发表事件：4 条来自 Springer 正式期刊页，6 条来自 CVPR 2026 官方论文集。证据等级固定为：

- **E1 正式出版**：出版社正式目录、官方论文集或论文正式页；
- **E2 官方决定**：会议 accepted list、decision 或 program，尚不等于正式论文集；
- **E3 双元数据一致**：Crossref 与 DBLP 等独立书目源一致，仍需回到出版方；
- **E4 聚合线索**：OpenAlex、Semantic Scholar 等聚合数据，只用于发现；
- **E5 预印本 / 自述**：arXiv、仓库、实验室、作者或社区页面，只能作为候选。

平台数量本身只扩大**发现覆盖**，绝不自动提高 E1–E5 证据等级。同一研究的身份按以下顺序核对：

E1–E5 只判断发表事实的证据强度，不代表论文水平。页面按“主张与证据、验证广度、真实场景、可复现性、版本增量”五项记录“证据充分 / 证据部分 / 暂不可判”；不能用 venue 名称、引用数或仓库 star 直接替论文打分。

```text
DOI → 去版本号 arXiv ID → 官方 proceedings / PMLR / DBLP / OpenReview / Anthology 等稳定 ID → 规范化标题 + 第一作者 + 首次公开年份
```

论文雷达负责具体论文、摘要和规范论文身份；学术追踪负责期刊/会议入口、版本与发表事件、来源覆盖、常用手法和编辑总结，不复制第二份论文正文。

## 4. 安装每日优化循环

每日循环依赖 Codex 执行，纯静态网页自身不能联网复核或修改项目。接收设备需要：

1. 安装 Codex Desktop，登录自己的账号，并至少启动一次；
2. 保持本项目位于长期不移动的目录；
3. 双击 `安装每日优化循环.bat`；
4. 完全退出并重新打开 Codex，在“自动化”页面确认 `Atlas 每日完整优化循环（可迁移）` 已启用。

任务默认每天本地时间 09:00 运行，使用安装脚本中配置的模型。若接收账号没有该模型权限，需要在 Codex 自动化页面换成该设备可用的模型。电脑关机、Codex 未运行、断网或额度不足时任务不会凭空执行；未完成项保存在 `.codex/atlas-maintenance-state.json` 的小型 schema v2 `pending` 中，下一轮先续做。

安装器只在接收者自己的 Codex 用户目录写入任务配置，并在覆盖同名任务前备份旧文件。它不会迁移账号、密钥或会话。本说明也不代表任务已经安装成功；必须在目标电脑实际运行安装器并到“自动化”页面确认。

## 5. 每日学术候选更新

建议每天运行一次，与 09:00 循环同频；默认回溯 30 天，不需要高频轮询：

```bash
python scripts/update_academic.py --days 30

# 只测试网络、解析、去重和统计，不写候选文件
python scripts/update_academic.py --days 30 --dry-run
```

脚本只读 `data/papers.json` 与 `data/academic_tracker.json`，唯一业务写入是 `data/academic_candidates.json`。该文件不被前端加载；其中每条记录都必须保持 `status: "needs_review"` 与 `manual_review_required: true`。候选只能由维护者打开原始来源后逐条审核，脚本永不直接合并 `data/details/`、旧 `data/models.json`、`data/index.json` 或 `data/academic_tracker.json`。

每天的学术结果固定输出：

- **事实摘要**：附一手来源和日期的可核验变化；
- **Atlas 理解**：明确标注为编辑判断，不冒充来源原话；
- **待核验**：证据缺口、失败来源和下一步应查位置。

不得猜测 JCR 分区、影响因子、会议录用率或当期 SCI/SCIE 状态；无法确认时保留 `unknown` 或“待核验”。

## 6. 迁移验收

- 10 个正式页面均能通过本地 HTTP 打开，无学习页或答题入口；
- 学术追踪三个 URL 可直达，刷新后保持当前视图；
- `data/academic_tracker.json` 显示 E1–E5 和 10 条已核验发表事件；
- `python scripts/test_update_academic.py` 与 `python scripts/validate_data.py` 通过；
- 搜索有明确按钮，滚动后结果仍可见，收藏为零时不显示孤立数字；
- 自动化页面显示每天 09:00 的任务；
- `.codex/atlas-maintenance-state.json` 是 schema v2，并如实保留真机未验证、公网未配置等 `pending`；
- 公网托管未配置时只称“本地/局域网站点”，不得称作公网已发布。
